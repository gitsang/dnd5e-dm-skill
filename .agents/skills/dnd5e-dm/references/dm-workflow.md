# DM 工作流

## 战役建立

先执行 `setup.md`：安装 CLI，创建 vault，确认规则版本，导入角色、SRD/CC、模组资料和批准来源。缺少来源时，把依赖该来源的裁定标记为不可确认。

## 跑团准备

读取 `campaign_log.md`、`world_state.json`、`module_index.json`、队伍状态和未解决线索。准备场景时保持模组目标，同时保留玩家选择。

## 实时 DM

只叙述玩家角色能感知的事实。机械结果先读 vault，再调用 CLI。多数场景以“你们怎么做？”结尾。

## 战斗

读取 `combat_state.json`。显示轮数、当前行动者、可见状态和剩余行动经济。每个声明先分类为 Action、Bonus Action、Reaction、Movement、Object Interaction 或免费交流。

## 规则裁定

先分类为工具强制、来源检查或 DM 判断。需要精确规则时引用本地批准来源；缺少来源时说明缺失，并提供 SRD/CC 替代或 homebrew 标记方案。

## 跑团后

更新确认事件、NPC 状态、阵营时钟、已发现线索、未解决钩子、资源变化和下次准备。玩家向总结不得泄露隐藏模组信息。
